🚀 Trae & GitHub 极简实操手册
本手册专为使用 Trae 编辑器的初学者编写，旨在帮助你快速掌握 Git 与 GitHub 的日常协作流程。

💡 核心概念秒懂

Git (本地时光机)：运行在你电脑上的系统，负责记录文件的每一次修改。

GitHub (云端代码库)：一个网站，用来存放你的 Git 记录，方便换电脑或与他人协作。

Commit (提交)：在本地保存一个“游戏存档点”。

Push (推送)：把本地的存档上传到 GitHub 云端。

Pull (拉取)：把 GitHub 云端最新的存档下载到本地。

🛠️ 阶段一：初次使用必做的“身份认证”

(仅在新电脑上第一次使用 Git 时需要执行)

打开 Trae 的终端（Terminal），依次输入并回车：

git config --global user.name "你的英文名或GitHub用户名"


git config --global user.email "你的GitHub注册邮箱"

这只是为了让 Git 知道“你是谁”，执行后不会有任何提示，代表设置成功。

🏗️ 阶段二：如何把本地新项目第一次传到云端？

当你写好了一个新项目（比如“星识”），想要传到云端时：

精准打开文件夹：在 Trae 中点击 文件 -> 打开文件夹，确保只打开你当前要上传的那个项目文件夹（千万不要打开外层的大文件夹！）。

在 GitHub 建空库：去 GitHub 网页点击 New 创建仓库（不要勾选 README 和 .gitignore），复制生成的 .git 链接。

执行初始化上传：在 Trae 终端依次输入以下 6 行命令：

git init                                    # 1. 初始化本地仓库
git add .                                   # 2. 将所有文件放进暂存区
git commit -m "第一次提交：项目初始化"          # 3. 创建第一个本地存档
git branch -M main                          # 4. 设定主分支叫 main
git remote add origin <https://github.com/你的用户名/项目名.git>  # 5. 绑定云端链接
git push -u origin main                     # 6. 正式推送到云端

🔄 阶段三：日常开发“黄金工作流” (敲命令版)

一旦完成了阶段二的绑定，你每天的工作就只剩下下面这些短命令了。

防冲突口诀：每天开工先拉取，每天完工推云端。

1. 开工前（下载云端最新代码）

如果项目只有你一个人写，且不换电脑，这步可省略。如果有别人修改过代码：

git pull

1. 完工后（上传你的今日进度）

在你写完代码、测试没问题后，依次输入这 3 条命令：

git add .
git commit -m "简短描述你今天修改了什么，比如：新增登录页面"
git push

🎨 阶段四：高手进阶“可视化操作” (免敲命令版)

其实在 Trae 里，你完全可以鼠标点点点完成所有日常操作，不用敲命令！

看 Trae 最左侧的侧边栏，点击第三个图标（源代码管理 / Source Control，图标长得像树枝分叉 ᛦ）：

暂存（等同于 git add .）：

看到“更改(Changes)”列表里有你修改过的文件。

悬浮在“更改”二字上，点击右侧的 + 号。文件会跑到“暂存的更改”列表中。

提交（等同于 git commit -m）：

在上方的文本框里输入你的备注（例如："修复了界面错位bug"）。

点击下方的蓝底白字大按钮 提交 (Commit)。

同步（等同于 git pull + git push）：

提交完成后，按钮会变成 同步更改 (Sync Changes)。

点击它！Trae 会自动先帮你把云端的拉下来，然后再把本地的推上去。

🚑 附录：常见急救指南

Q1: 糟糕！我在外层大文件夹执行了 git init，把其他作业都卷进去了怎么办？
自救方案：在错放了 Git 的那个目录终端里，输入 rm -r -Force .git (Windows) 或 rm -rf .git (Mac/Linux) 销毁记录。然后用 Trae 重新打开正确的单体项目文件夹。

Q2: 提示 "Permission denied" 或一直让我输入账号密码？
原因：GitHub 网络抽风或凭证过期。
自救方案：注意看屏幕弹出的浏览器窗口，允许 "Git Credential Manager" 授权登录。或者如果开了网络代理（梯子），尝试开关一下代理再重试。

Q3: 遇到代码冲突 (Merge Conflict) 怎么办？
原因：你和你的朋友修改了同一行代码。
自救方案：Trae 会高亮显示冲突的代码块。你会看到 Accept Current Change (保留你的修改)、Accept Incoming Change (保留朋友的修改) 等选项。点击你想要的那个，保存文件，然后重新 add, commit, push。
